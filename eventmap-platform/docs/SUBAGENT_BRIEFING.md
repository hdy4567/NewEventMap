### 🚀 Critical Architecture Update
The system has been stabilized with a **Dual-Port Strategy** to eliminate conflict between the Frontend and Backend:
1.  **Vite Frontend (Port 9005)**: Standard development port for HMR, Hot Reloading, and Debugging.
2.  **C# Backend Bridge (Port 9091)**: WebSocket API, AI Agent, and Sync services now listen on 9091.
3.  **Port Conflict (9005)**: The C# server no longer kills processes on 9005, allowing Vite to remain active during development.

- **Vite Integration**: `worker_bridge.js` now instantiates the worker using `new URL('./ai_neural_worker.js', import.meta.url)` to ensure correct module resolution.
- **Transformer.js**: Local model loading (`Xenova/paraphrase-multilingual-MiniLM-L12-v2`) remains active with IndexedDB persistence.

#### 2. Directory & File Cleanup
- **Token Efficiency**: Moved all `.log` files, temporary `.spec.js`, and debug `.png` images from the `frontend-web` root to a hidden `.logs/` or sub-folders.
- **AppConfig**: Backend port is now centrally managed in `AppConfig.cs` (Port 9091).

#### 3. Module Unification
- **State.js**: Added `loadStore` and `saveToDB` logic for IndexedDB.
- **App.js**: Main entry point for `startApp()`, coordinating Map, Worker, and WebSocket initialization.

### ⚠️ Current Risks
1. **Z-Index Layering**: Multiple floating panels (AI Monitor, Locker, Side-Apps) must be managed carefully. Current range: `1000-3000`.
2. **WebSocket Port Mismatch**: If the backend is not on 9091, the frontend will show a connection failure alert.
3. **Browser Cache**: Renaming the worker only helps partially; users MUST clear "Application Site Data" if SyntaxErrors persist.
